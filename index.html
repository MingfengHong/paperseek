PaperSeekLLM based Literature Search Agent

用自然语言发起文献检索，自动迭代查询、扩展引用、排序结果，并导出可复核的候选论文列表。

GitHub 仓库

自然语言检索

输入中文或英文研究问题，由 LLM 生成适合 OpenAlex、arXiv、Semantic Scholar、PubMed、Google Scholar、计算机顶会、Crossref 或 WoS Starter 的检索查询。

可观察工作流

实时查看查询生成、数据源请求、检索式调整、结果排序、引用扩展和系统日志；排序阶段会拆分显示多路召回、embedding、RRF、LLM 批量打分和摘要补全等子步骤。

学科领域限制

可在 Research Question 下方选择数据源和 Source Filter；OpenAlex、WoS、arXiv 使用原生硬过滤，其它数据源使用领域文本提示辅助 LLM 构建检索式。

中英文界面

Web UI 支持 EN / 中文切换；在线版和开源自托管版使用一致的主要交互结构。

引用扩展与图谱

基于 OpenAlex 扩展高匹配论文的参考文献和被引论文，并用箭头展示引用方向。

可复核导出

在 Results 页面筛选、勾选、排序论文，导出包含元数据和评分理由的 CSV；系统会至少保留前 50 条结果，并在高分候选更多时展示全部 5 分及以上论文。

Agent 与 MCP 集成

可复制自包含 Skill，或安装 paperseek[mcp] 启动 stdio MCP Server，让支持 MCP 的 AI agent 调用检索、诊断和历史工具。

选择你的入口 ​

PaperSeek 的社区文档分为两个主要入口：

  • 开源自托管版：阅读 开源版完整用户手册，适合本地运行、Docker/VPS/ModelScope 创空间部署、配置自己的 LLM Key、数据源 Key 和默认 Discipline Fields。
  • 在线体验版：阅读 在线体验版用户手册，适合直接访问 paperseek.xyz 试用完整 Web UI。在线版支持 PaperSeek Service、Third-party Service（ModelScope、OpenRouter、Hugging Face）和 Bring your own Key / BYOK 三种模式，并提供与自托管版一致的源特定过滤逻辑；登录和第三方授权会影响 PaperSeek Service、Third-party Service 与云端历史。
  • Agent 集成：用户手册中的 Agent Skill 和 MCP Server 章节说明如何把 PaperSeek 接入支持 Skill 或 MCP 的 AI agent。

部署 Docker、Docker Compose、Vercel 或 ModelScope 创空间时，阅读 部署指南。

社区维护方式 ​

文档源码保存在 main 分支的 docs/ 目录。GitHub Actions 构建后会把静态产物发布到 gh-pages 分支。gh-pages 只是 GitHub Pages 发布分支，不作为社区开发分支。
